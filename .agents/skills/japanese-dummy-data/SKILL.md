When generating sample data or placeholder content for web layouts, strictly follow these rules to ensure high-quality Japanese mockups:

1. **NO "Lorem Ipsum":** Never use "Lorem Ipsum" or any English placeholder text. Always use natural Japanese.
2. **Context-Aware Content:** Generate text that matches the context of the UI component.
   - *Example:* If it's a "News" section, write a realistic corporate announcement (e.g., "夏季休業のお知らせ", "新サービス開始のお知らせ").
   - *Example:* If it's a "User Review" section, write realistic Japanese customer feedback.
3. **Realistic Names:** When placeholder names are needed, generate realistic Japanese names (e.g., 山田 太郎, 佐藤 花子, 株式会社サンプル). Do NOT use literal translations of Western names (e.g., ジョン・ドウ).
4. **Appropriate Length:** Adjust the length of the Japanese text to naturally fit the designed HTML/CSS container without breaking the layout.
5. **Fictional Contact Data:** When contact information or dates are needed, always use clearly fictional data following Japanese formats.
   - Dates: Use realistic but clearly fictional dates (e.g., 2024年4月1日). Avoid using today's actual date.
   - Phone numbers: Use fictional numbers following Japanese format (e.g., 03-0000-0000).
   - Email addresses: Use fictional addresses (e.g., info@example.co.jp).
6. **Contextual Dummy Images:** Always use Unsplash or equivalent high-quality placeholder image services. Choose themes or keywords that perfectly match the context of the section (e.g., business, cafe, portrait) to ensure the visuals align with the generated Japanese text. If Unsplash is unavailable, use picsum.photos as an alternative. Do not use blank or gray placeholder boxes.
